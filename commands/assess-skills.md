# /assess-skills

Evaluate your current skills and get recommendations for improvement.

## Usage

```
/assess-skills
```

## Description

This command helps you assess your technical skills across the 7 major specializations and provides personalized recommendations for growth.

## Assessment Format

For each specialization, you'll evaluate:

1. **Knowledge Level** (1-5 scale)
   - 1: No experience
   - 2: Basic understanding
   - 3: Intermediate proficiency
   - 4: Advanced expertise
   - 5: Expert/specialist level

2. **Practical Experience** (1-5 scale)
   - Hours of hands-on practice
   - Projects completed
   - Production systems deployed

3. **Technology Coverage**
   - Core technologies mastered
   - Advanced topics explored
   - Tools proficiency

## Specializations Evaluated

- **Frontend Developer** - UI frameworks, styling, state management
- **Backend Developer** - APIs, databases, server architecture
- **DevOps Engineer** - Containerization, orchestration, infrastructure
- **AI/ML Engineer** - ML models, deep learning, LLMs
- **Mobile Developer** - iOS, Android, cross-platform
- **Data Engineer** - Pipelines, warehouses, analytics
- **System Architect** - Design, scalability, patterns

## What You'll Receive

1. **Skills Assessment Report**
   - Your current proficiency level
   - Strength areas (4-5 rating)
   - Development areas (1-3 rating)
   - Comparison with industry standards

2. **Personalized Recommendations**
   - Priority learning areas
   - Suggested next steps
   - Timeline to next level
   - Resources and courses
   - Hands-on project ideas

3. **Growth Plan**
   - 3-month goals
   - 6-month milestones
   - 1-year objectives
   - Skill combinations for roles

4. **Career Path Suggestions**
   - Best-fit specializations
   - Cross-specialization synergies
   - Market demand for your skills
   - Salary/career progression info

## Example Assessment

**Current Profile:**
- Frontend: 4/5 (Expert in React)
- Backend: 2/5 (Basic Node.js)
- System Design: 2/5 (Limited experience)

**Recommendation:**
→ Focus on Backend fundamentals → System Architecture
→ Become Full Stack developer → Tech Lead
→ Timeline: 12-18 months

## Tips for Accurate Assessment

- Be honest about your experience
- Consider both theoretical and practical knowledge
- Include recent and past projects
- Account for technologies you've mastered vs. dabbled in
- Think about depth, not just breadth

## Follow-up

After assessment, you can:
- `/learn-path` to start structured learning
- Ask agents about specific skill gaps
- Request hands-on project ideas
- Get resource recommendations
